## 第一部分：項目概覽

### 1.1 項目定位

BeeJAI（Bee仔升學）是一個專為香港家長設計的升學資訊平台，提供幼稚園、小學、中學及國際學校的全面資訊，協助家長為子女規劃最佳升學路徑。

### 1.2 核心價值主張

| 價值點 | 描述 |
|--------|------|
| **全面資訊** | 涵蓋全港 2000+ 學校資料，包括幼稚園、小學、中學、國際學校 |
| **智能篩選** | 12 維度篩選條件，快速找到合適學校 |
| **升學攻略** | 專業攻略文章，涵蓋 IB 選校、內地來港專才、小一入學等熱門話題 |
| **申請追蹤** | 心儀學校收藏、申請進度追蹤、截止日期提醒 |
| **實用工具** | 學費計算機、Playgroup 推薦等實用工具 |

## 第三部分：學校資訊導入計劃

### 3.1 數據來源分析

香港學校資訊可從以下渠道獲取：

| 數據來源 | 數據類型 | 更新頻率 | 獲取方式 |
|----------|----------|----------|----------|
| **教育局官網** | 學校名冊、基本資料 | 每年更新 | 公開 CSV/Excel 下載 |
| **學校概覽** | 學校詳細資訊 | 每年更新 | 網頁爬取 |
| **各校官網** | 學費、課程、入學要求 | 不定期 | 網頁爬取 |
| **升學網站** | 排名、評價、面試題目 | 即時 | API 或爬取 |

### 3.2 數據導入策略（最低 Credit 方案）

**階段一：基礎數據導入（預計 2-3 小時）**

使用教育局公開數據作為基礎，無需 AI 處理：

```
步驟 1: 下載教育局學校名冊 CSV 文件
步驟 2: 編寫 Python 腳本解析 CSV，轉換為資料庫格式
步驟 3: 批量導入學校基本資料（名稱、地址、電話、類型、區域）
步驟 4: 驗證數據完整性
```

**階段二：補充數據爬取（預計 4-6 小時）**

使用傳統爬蟲技術獲取補充資訊：

```
步驟 1: 編寫 Playwright/Puppeteer 爬蟲腳本
步驟 2: 爬取學校官網獲取學費、課程資訊
步驟 3: 爬取升學網站獲取排名、評價數據
步驟 4: 數據清洗和格式化
步驟 5: 增量更新資料庫
```

**階段三：AI 輔助數據增強（可選，按需使用）**

僅在必要時使用 AI 處理：

| 使用場景 | AI 任務 | 預計 Token 消耗 |

| 數據標準化 | 統一地址、學費格式 | 批量處理，低消耗 |

### 3.3 資料庫結構設計

```sql
-- 學校主表
CREATE TABLE schools (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(255) NOT NULL,
  name_en VARCHAR(255),
  type ENUM('kindergarten', 'primary', 'secondary', 'international'),
  district VARCHAR(50),
  address TEXT,
  phone VARCHAR(20),
  website VARCHAR(255),
  founded_year INT,
  gender ENUM('coed', 'boys', 'girls'),
  religion VARCHAR(50),
  sponsoring_body VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 學費表
CREATE TABLE school_fees (
  id INT PRIMARY KEY AUTO_INCREMENT,
  school_id INT REFERENCES schools(id),
  academic_year VARCHAR(10),
  fee_type VARCHAR(50),
  amount DECIMAL(10,2),
  currency VARCHAR(3) DEFAULT 'HKD'
);

-- 課程表
CREATE TABLE school_programs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  school_id INT REFERENCES schools(id),
  program_type VARCHAR(50), -- DSE, IB, IGCSE, etc.
  description TEXT
);
```

### 3.4 數據導入腳本範例

```python
#!/usr/bin/env python3
"""
學校數據導入腳本 - 從教育局 CSV 導入基礎數據
"""

import csv
import json
from pathlib import Path

def parse_edb_csv(csv_path: str) -> list:
    """解析教育局學校名冊 CSV"""
    schools = []
    with open(csv_path, 'r', encoding='utf-8') as f:
        reader = csv.DictReader(f)
        for row in reader:
            school = {
                'name': row.get('學校名稱', ''),
                'name_en': row.get('School Name', ''),
                'type': map_school_type(row.get('學校類別', '')),
                'district': row.get('地區', ''),
                'address': row.get('地址', ''),
                'phone': row.get('電話', ''),
                'website': row.get('網址', ''),
            }
            schools.append(school)
    return schools

def map_school_type(type_str: str) -> str:
    """映射學校類型"""
    type_mapping = {
        '幼稚園': 'kindergarten',
        '小學': 'primary',
        '中學': 'secondary',
        '國際學校': 'international',
    }
    return type_mapping.get(type_str, 'primary')

def generate_seed_sql(schools: list) -> str:
    """生成 SQL 插入語句"""
    sql_lines = []
    for school in schools:
        sql = f"""
INSERT INTO schools (name, name_en, type, district, address, phone, website)
VALUES ('{school['name']}', '{school['name_en']}', '{school['type']}', 
        '{school['district']}', '{school['address']}', '{school['phone']}', 
        '{school['website']}');
"""
        sql_lines.append(sql)
    return '\n'.join(sql_lines)

if __name__ == '__main__':
    # 使用範例
    schools = parse_edb_csv('edb_schools.csv')
    sql = generate_seed_sql(schools)
    Path('seed_schools.sql').write_text(sql)
    print(f'Generated SQL for {len(schools)} schools')
```

## 第五部分：效率優化方案

### 5.1 最低 Credit 開發策略

為了以最低的 Credit 達到最高效率，建議採用以下策略：

**原則一：優先使用現有模板和組件**

| 場景 | 低效做法 | 高效做法 |
|------|----------|----------|
| UI 組件 | 從零開始編寫 | 使用 shadcn/ui 現有組件 |
| 頁面佈局 | 每頁獨立設計 | 複用 DashboardLayout 等模板 |
| 表單驗證 | 手動編寫邏輯 | 使用 zod + react-hook-form |

**原則二：批量處理取代逐一處理**

| 場景 | 低效做法 | 高效做法 |
|------|----------|----------|
| 數據導入 | 逐條插入資料庫 | 批量 SQL INSERT |
| 文件編輯 | 多次單獨修改 | 一次性多處編輯 |
| API 調用 | 多次單獨請求 | 批量請求或緩存 |

**原則三：減少不必要的 AI 調用**

| 場景 | 需要 AI | 不需要 AI |
|------|---------|-----------|
| 數據格式轉換 | ✗ | 使用腳本處理 |
| 簡單文字修改 | ✗ | 直接編輯 |
| 複雜邏輯生成 | ✓ | - |
| 創意內容撰寫 | ✓ | - |
